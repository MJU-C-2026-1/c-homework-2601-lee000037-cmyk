# 💻 [2026-01] 나만의 C언어 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 2024시즌 구자욱 선수의 타격성적 분석

<br>

## 2. 시나리오 (5~10줄)
* 이 프로그램은 구자욱 선수의 커리어하이시즌인 2024년 타격성적을 분석하는 '야구 기록 분석 시스템'입니다.
사용자는 구자욱 선수의 기초정보 및 2024년 타격성적을 입력합니다. 시스템은 이를 바탕으로 구자욱 선수의 성적을 분석하고, 성적에 따라 선수의 등급(예시 : MVP급, 골든글러브급, 올스타급, 주전급, 백업급, 2군급)을 부여합니다. 기초정보로는 선수의 이름과 등번호 등이 취급될 예정이며, 선수의 성적으로는 기초적인 출전경기숫자와 타율, 홈런부터 출루율과 장타율, 그리고 그 2개를 합산한 OPS등을 다룰 예정입니다.
또한, 복합적인 조건(예시: 홈런 20개 이상, 타율 3할 이상)을 만족하면 '엘리트' 칭호를 지급하는 특별 취급 로직이 포함 됩니다. 최종적으로는 사용자가 메뉴판을 통해 원하는 기능을 자유롭게 선택하고, 여러개의 기록을 동시에 관리하여 구자욱 선수의 2024년 기록을 조회할 수 있도록 전체 시스템을 완성할 계획입니다. 
  
<br>

## 3. 예상 기능 및 메뉴 (최소 5개)

1. 구자욱 선수의 기초정보 입력
2. 구자욱 선수의 타격성적 입력
3. 야구 스탯의 등급 산정
4. 칭호 판별
5. 구자욱 선수의 스탯 전체 출력
0. 프로그램 종료

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
  * 사용자로부터 구자욱 선수의 기초 정보를 출력합니다.
  * 사용자로부터 구자욱 선수의 타격 성적을 출력합니다.
  * 구자욱 선수의 이니셜(문자), 선수 등번호(정수), 타격 성적(실수)을 입력받는 화면을 구현했습니다. 
  * 사칙연산을 활용해 구자욱 선수의 2024년도 'OPS'를 계산하여 출력했습니다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:** "C언어 4단계 과제 로드맵(입출력 ➔ 조건문 ➔ 반복문 ➔ 배열/함수)에 적합한 구자욱 선수의 가치 평가 기획 요청"
    * **적용 내용:** 단순한 스탯 출력기를 넘어, '스탯 입력(1차) ➔ 스탯 등급 산정(2차) ➔ 성적 칭호 시스템(3차) ➔ 구자욱의 가치 판별(4차)'로 이어지는 논리적인 시스템 확장 로드맵과 예상 메뉴를 확립함.
      
 2. **내용 2**
    * **프롬프트 요약:** "1차 과제 필수 요건(변수 5개, 자료형 3개 이상)을 충족하기 위한 캐릭터 초기 스탯 데이터 구조 모델링 논의"
    * **적용 내용:** 포괄적인 의미의 '공격력' 변수 하나를 사용하는 대신, 데이터 특성에 맞춰 문자형(`char` - 이니셜), 정수형(`int` - 등번호, 누적 성적), 실수형(`float` - 비율 성적)으로 자료형을 명확히 분리하여 입력 데이터의 구조적 완성도를 높임.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 실수형 데이터(비율 스탯)를 출력할 때 소수점이 너무 짧게 나옴.
     * **해결:** 강의자료와 구글링, AI 검색을 통해 `printf("%.3f", weapon_power);`와 같이 서식 지정자를 수정하여 소수점 셋째 자리까지만 출력되도록 개선함.
  2. **문제 2:** 깃허브에 pdf와 jpg파일 첨부 방법을 모름.   
     * **해결:** 강의자료와 AI 검색을 통해 README.md에서 PDF 링크 걸기, 저장소(Repository)에 직접 올려서 보여주기로 해결함.
  3. **문제 3:** 깃허브 코드스페이스 main.c에 작성한 코드 실행 방법을 모름.
     * **해결:** AI 검색을 통해 컴파일 및 실행 방법을 찾음.
  4. **문제 4:** 문자열 배열이 출력이 안됌.
     * **해결:** 강의자료와 AI 검색을 통해 문자열 배열은 & 기호를 붙이지 않는다는 사실 확인.
